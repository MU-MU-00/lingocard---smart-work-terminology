# LingoCard 使用指南

## 一、在电脑上打开应用

1. 用 Cursor 打开项目文件夹。
2. 终端里执行（在项目根目录）：
   ```bash
   npm run dev
   ```
   或：`npm.cmd run dev`
3. 浏览器会自动打开 **http://localhost:3000**，或手动打开这个地址。

---

## 二、日常怎么用

| 操作 | 做法 |
|------|------|
| **添加术语** | 在首页输入框输入英文或中文术语（如 API、KPI）→ 回车或点右侧 ➕ → 等待生成卡片 → 下滑保存到分组，或右滑选择分组后保存。 |
| **管理分组** | 左侧橙色栏是分组，点击切换；底部 ➕ 可新建分组。在分组里可左滑列表项露出删除/收回。 |
| **看/改术语** | 点分组 → 点某条术语进入大卡片 → 点右下角编辑按钮可改名称翻译、名词解释、例句等 → 点保存。 |
| **复习** | 首页点「Daily Review」→ 选全部或某个分组 → 按提示选答案，系统会记录并安排下次复习。 |
| **发音** | 在卡片或列表里点 🔊 图标，用浏览器朗读术语。 |

数据会**自动保存**：先存浏览器，若已配置 Supabase 会同步到云端，无需手动点保存。

---

## 三、在手机上用（可选）

- **方式 A**：把项目部署到 **Vercel** 或 **GitHub Pages**（见 [DEPLOY.md](DEPLOY.md)），用手机浏览器打开部署后的链接；数据在 Supabase 里，换设备也能看到。
- **方式 B**：电脑运行 `npm run dev`，手机和电脑连**同一 WiFi**，在手机浏览器输入终端里显示的 **Network 地址**（如 `http://192.168.x.x:3000`），可临时在手机访问。

---

## 四、更新代码并推到 GitHub

1. 在 Cursor 里改代码 → **Ctrl + S** 保存。
2. 打开 **GitHub Desktop** → 看到改动列表 → 写一句 **Summary** → **Commit to main** → **Push origin**。
3. 若已用 Vercel 部署，推送后会自动重新部署，线上/手机打开的链接会更新。

---

## 五、环境与配置

- **.env.local**：放豆包 API 密钥（`GEMINI_API_KEY`）和 Supabase（`VITE_SUPABASE_URL`、`VITE_SUPABASE_ANON_KEY`）。不要提交到 GitHub。
- **Supabase**：配置好后术语和分组会同步到云端；未配置或安装失败时只用浏览器本地存储，刷新不丢数据。

按以上步骤即可正确、持续使用你的应用。
